- Предусмотреть механизм генерации уникального серийного номера на базе 96-bit UID, на базе него сформировать ID для MQTT, каким-то образом
  передать его во внешний мир, чтобы можно было создать личный кабинет на сервере MQTT


- Проверка работоспособности связи с симками разных операторов, при необходимости, установка APN, соответствующего оператора (Мегафон, Билайн, МТС, Теле2)
  Если не заработает, то задание правильного apn, в зависимости от оператора
- Отключать регулирование, при возникновении ошибок по контакторам. Контактор направления (КМ8 для четырехступ., КМ9 для пятиступ.) контролировать
  виртуально, а именно, если после включения заданной ступени напряжение не понизилось, а повысилось (либо, не повысилось, а понизилось), то считать,
  что контактор неисправен, прекратить регулирование (перейти в транзит)
- Передавать информацию на сервер о каждом переключении каждого контактора. Необходимо, для изучения аварийных ситуаций, когда все сгорело, а из-за чего непонятно.

- После включения устройства включать лампочку на сервере
- Передвать источник сброса.
  Это может быть включение по питанию, сброс по ватчдогу (два вида), сброс по пину reset, программный сброс и ещё какой-то lowpower
- При обновлениеи проверять наличие симки. Если нет - сразу выходить. Если есть пытаться обновиться до последнего.
- Выводить на дисплей информацию о версию (4 или 5 ступеней)
- На MQTT передавать информацию не только о версии прошивки, но и о количестве ступеней
- Добавить команду включения/отключения засылки данных
